# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между слоями:

Model — слой данных, отвечает за хранение и изменение данных.  
View — слой представления, отвечает за отображение данных на странице.  
Presenter — презентер содержит основную логику приложения и отвечает за связь представления и данных.

В текущей итерации проекта реализованы **все три слоя**: Model, View и Presenter.

- **Model** хранит данные и эмитит события при любых изменениях состояния.
- **View** отвечает за отрисовку DOM и преобразует действия пользователя в события.
- **Presenter** подписывается на события от Model и View и содержит основную логику приложения.

Ключевое правило: представление перерисовывается только:
1) при обработке события от модели (данные изменились);
2) при открытии модального окна.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные приложения

### Описание типов данных

#### TPayment

Тип для выбора способа оплаты.

```typescript
type TPayment = "card" | "cash";
```

#### IProduct

Интерфейс для описания товара в каталоге.

Поля:

- `id: string` — уникальный идентификатор товара
- `description: string` — описание товара
- `image: string` — путь к изображению товара
- `title: string` — название товара
- `category: string` — категория товара
- `price: number | null` — цена товара (null если товар недоступен)

#### IBuyer

Интерфейс для описания данных покупателя при оформлении заказа.

Поля:

- `payment: TPayment` — способ оплаты ("card" или "cash")
- `email: string` — электронная почта покупателя
- `phone: string` — номер телефона покупателя
- `address: string` — адрес доставки

#### IProductsResponse

Интерфейс для ответа сервера при получении каталога товаров.

Поля:

- `total: number` — общее количество товаров в каталоге
- `items: IProduct[]` — массив товаров

#### IOrder

Интерфейс для данных заказа, отправляемых на сервер при оформлении покупки.

Поля:

- `payment: TPayment` — выбранный способ оплаты
- `email: string` — email покупателя
- `phone: string` — телефон покупателя
- `address: string` — адрес доставки
- `total: number` — общая стоимость заказа
- `items: string[]` — массив ID товаров в заказе

#### IOrderResponse

Интерфейс для ответа сервера при успешном оформлении заказа.

Поля:

- `orderId: string` — уникальный ID оформленного заказа
- `total: number` — общая сумма заказа

## Модели данных

Модели данных отвечают за хранение и управление данными приложения. Они полностью независимы от других компонентов и не содержат логики отображения.

### Класс Products (Каталог товаров)

**Назначение:** Хранение всех доступных товаров в каталоге и управление выбранным товаром для подробного просмотра.

**Файл:** `src/components/models/Products.ts`

**Конструктор:**

- Параметры: нет
- Инициализирует пустой массив товаров и null для выбранного товара

**Поля:**

- `items: IProduct[]` — массив всех доступных товаров в каталоге
- `selectedItem: IProduct | null` — товар, выбранный для детального просмотра (null если товар не выбран)

**Методы:**

- `setItems(items: IProduct[]): void` — сохранить массив товаров в каталог
- `getItems(): IProduct[]` — получить все товары из каталога
- `getProductById(id: string): IProduct | null` — найти товар по его ID (возвращает null если товар не найден)
- `setSelectedProduct(item: IProduct): void` — выбрать товар для подробного просмотра
- `getSelectedProduct(): IProduct | null` — получить текущий выбранный товар (null если товар не выбран)

### Класс Cart (Корзина)

**Назначение:** Управление товарами, которые пользователь выбрал для покупки, расчет стоимости и количества.

**Файл:** `src/components/models/Cart.ts`

**Конструктор:**

- Параметры: нет
- Инициализирует пустую корзину

**Поля:**

- `items: IProduct[]` — массив товаров, находящихся в корзине

**Методы:**

- `addProduct(product: IProduct): void` — добавить товар в корзину (проверяет наличие товара и что цена не null)
- `removeProduct(productId: string): void` — удалить товар из корзины по его ID
- `getProducts(): IProduct[]` — получить все товары в корзине
- `getTotalPrice(): number` — получить общую стоимость всех товаров в корзине
- `getItemCount(): number` — получить количество товаров в корзине
- `isProductInCart(productId: string): boolean` — проверить находится ли товар в корзине
- `clearCart(): void` — очистить корзину полностью

### Класс Order (Данные покупателя и заказа)

**Назначение:** Хранение и управление данными покупателя при оформлении заказа, включая валидацию данных.

**Файл:** `src/components/models/Order.ts`

**Конструктор:**

- Параметры: нет
- Инициализирует пустые данные покупателя

**Поля:**

- `data: IBuyer` — объект с данными покупателя (payment, email, phone, address)

**Методы:**

- `setOrderData(data: Partial<IBuyer>): void` — сохранить или обновить данные покупателя (использует Partial для частичного обновления без затирания других полей)
- `getOrderData(): IBuyer` — получить все сохраненные данные покупателя
- `clearOrderData(): void` — очистить все данные покупателя (сбросить в пустые значения)
- `validateOrderData(): { [key: string]: string }` — валидировать данные заказа, возвращает объект с ошибками (пустой объект если ошибок нет). Проверяет валидность email, телефона, адреса и выбор способа оплаты.

## Слой коммуникации

Слой коммуникации отвечает за взаимодействие приложения с API сервера.

### Класс Server

**Назначение:** Выполнение запросов к API сервера для получения товаров и отправки заказов.

**Файл:** `src/components/models/Server.ts`

**Конструктор:**

- Параметры: `api: IApi` — экземпляр класса Api для выполнения HTTP запросов
- Сохраняет переданный экземпляр API для использования в методах класса


**Методы:**

- `async getProducts(): Promise<IProduct[]>`
  - Выполняет GET запрос на эндпоинт `/product/`
  - Возвращает промис с массивом товаров
  - В случае ошибки выводит сообщение об ошибке в консоль и пробрасывает ошибку дальше

- `async createOrder(order: IOrder): Promise<IOrderResponse>`
  - Выполняет POST запрос на эндпоинт `/order/`
  - Отправляет данные заказа на сервер
  - Возвращает промис с ответом сервера (ID и сумма заказа)
  - В случае ошибки выводит сообщение об ошибке в консоль и пробрасывает ошибку дальше

## Слой представления (View)

Слой View отвечает за работу с DOM: находит элементы разметки, обновляет их и подписывается на пользовательские события.
Представления **не хранят данные** и **не содержат бизнес-логики** — они только эмитят события, которые обрабатывает презентер.

### Класс Modal (Модальное окно)

**Назначение:** Управление отображением модального окна и его содержимым.

**Файл:** `src/components/views/Modal.ts`

**Конструктор:**
- Параметры: `container: HTMLElement`, `events: IEvents`
- Находит элементы модалки и подписывается на события закрытия (крестик, клик по оверлею).

**Поля:**
- `container: HTMLElement` — корневой элемент модалки
- `content: HTMLElement` — контейнер для контента
- `closeButton: HTMLButtonElement` — кнопка закрытия

**Методы:**
- `setContent(content: HTMLElement): void` — устанавливает контент модалки (полностью заменяет предыдущий)
- `open(): void` — открывает модалку (добавляет модификатор `modal_active`)
- `close(): void` — закрывает модалку (удаляет модификатор)

### Класс Header

**Назначение:** Отображение шапки приложения и счётчика товаров в корзине.

**Файл:** `src/components/views/Header.ts`

**Конструктор:**
- Параметры: `container: HTMLElement`, `events: IEvents`

**Поля:**
- `basketButton: HTMLButtonElement` — кнопка открытия корзины
- `counter: HTMLElement` — счётчик товаров в корзине

**Методы:**
- `counter: number` (setter) — обновляет значение счётчика корзины

### Класс Gallery

**Назначение:** Отображение каталога товаров на главной странице.

**Файл:** `src/components/views/Gallery.ts`

**Конструктор:**
- Параметры: `container: HTMLElement`, `events: IEvents`

**Поля:**
- `container: HTMLElement` — корневой элемент галереи

**Методы:**
- `items: HTMLElement[]` (setter) — устанавливает готовую разметку каталога

### Класс Basket

**Назначение:** Отображение списка товаров в корзине и общей стоимости.

**Файл:** `src/components/views/Basket.ts`

**Конструктор:**
- Параметры: `container: HTMLElement`, `events: IEvents`

**Поля:**
- `list: HTMLElement` — список товаров
- `total: HTMLElement` — элемент общей стоимости
- `button: HTMLButtonElement` — кнопка оформления заказа

**Методы:**
- `items: HTMLElement[]` (setter) — устанавливает готовые элементы корзины
- `total: number` (setter) — устанавливает итоговую сумму
- `disabled: boolean` (setter) — управляет доступностью кнопки оформления

### Класс Success

**Назначение:** Экран успешного оформления заказа.

**Файл:** `src/components/views/Success.ts`

**Конструктор:**
- Параметры: `container: HTMLElement`, `events: IEvents`

**Поля:**
- `total: HTMLElement` — отображение итоговой суммы заказа
- `closeButton: HTMLButtonElement` — кнопка закрытия

**Методы:**
- `setTotal(total: number): void` — устанавливает итоговую сумму заказа

### Карточки товара (Card)

Для карточек используются три шаблона, поэтому реализована иерархия классов:
- `Card` — базовый класс с общим функционалом
- `CardCatalog` — карточка в каталоге
- `CardPreview` — карточка в предпросмотре
- `CardBasket` — карточка в корзине

`Card` использует `categoryMap` из `src/utils/constants.ts` для корректной раскраски категории.

**Общий интерфейс карточек:**
- `display(product: IProduct): HTMLElement` — отрисовка карточки товара
- события пользовательских действий передаются через колбэки или EventEmitter (в зависимости от типа карточки)

### Формы оформления заказа (Form)

Формы имеют общего родителя:
- `Form` — базовый класс формы (сбор данных, обработка submit/change, отображение ошибок)
- `OrderForm` — шаг 1 (оплата + адрес)
- `ContactsForm` — шаг 2 (почта + телефон)

**Общий интерфейс форм:**
- формы не хранят данные и не валидируют их
- изменения полей эмитятся событием `form:change`
- отправка формы эмитит `form:submit` или `order:contact`

## Презентер (Presenter)

Презентер содержит основную логику приложения и связывает Model и View.
Реализация находится в `src/components/Presenter.ts`, запуск — в `src/main.ts`.

**Основные обязанности презентера:**
- загрузка каталога с сервера и сохранение в модель;
- открытие модальных окон (предпросмотр, корзина, формы, успех);
- обработка событий от View (клики, сабмиты, изменения форм);
- обработка событий от Model (изменения каталога/корзины/данных покупателя);
- отправка заказа на сервер и очистка корзины/данных покупателя при успехе.

## События приложения

Приложение построено на событиях: любое действие пользователя и любое изменение данных эмитит событие.
Все события обрабатываются в презентере.

### События моделей (Model → Presenter)

- `catalog:changed` — каталог товаров обновлён
- `product:selected` — выбран товар для предпросмотра
- `cart:changed` — корзина изменилась
- `order:changed` — изменились данные покупателя

### События представлений (View → Presenter)

- `basket:open` — пользователь открыл корзину
- `basket:toggle` — кнопка действия в предпросмотре (купить / удалить)
- `order:open` — начать оформление заказа
- `form:change` — изменение данных в формах
- `form:submit` — переход ко второму шагу оформления
- `order:contact` — завершение оформления (оплата)
- `modal:close` — закрытие модального окна
- `success:close` — закрытие окна успеха

## Точка входа приложения / main.ts

Назначение:
`main.ts` — точка входа приложения. Здесь запускается презентер и связываются слои Model и View.

В `main.ts`:
- подключаются стили;
- создаётся экземпляр `Presenter`;
- вызывается `init()` для загрузки каталога и запуска приложения.
